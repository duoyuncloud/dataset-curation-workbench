Update the project so it can be deployed as a single public web app.

Goal:
Users should be able to open one public URL and use the full SFT Dataset Curation Workbench. They should NOT need to run backend and frontend separately.

---

# Deployment target

Use a single-service deployment model:

* FastAPI serves all API routes under `/api/...`
* FastAPI also serves the React production build under `/`

In production:

* user opens one URL
* React frontend loads
* frontend calls `/api/...`
* no localhost URLs are exposed to users

---

# Keep development mode unchanged

Do NOT break local development.

Local dev should still support:

* backend on `localhost:8000`
* frontend on `localhost:5173`

Use Vite proxy or environment variable for local API calls.

---

# Backend changes

Update FastAPI routes so all API endpoints are prefixed with `/api`.

Example:

* `/api/datasets/upload`
* `/api/datasets/{dataset_id}/stages`
* `/api/datasets/{dataset_id}/apply-filters`
* `/api/datasets/{dataset_id}/export`

Add static frontend serving in production.

After React is built into `frontend/dist`, FastAPI should serve it:

* `/` returns React app
* unknown non-API routes return React app for client-side routing
* `/api/...` remains backend API

Use `fastapi.staticfiles.StaticFiles`.

---

# Frontend changes

Update frontend API client so it can work in both dev and production.

Use:

* development: `VITE_API_BASE_URL=http://localhost:8000/api`
* production: default to `/api`

Do not hardcode localhost in production.

---

# Build scripts

Add root-level scripts:

1. `scripts/build_frontend.sh`

Should:

* cd frontend
* npm install if needed
* npm run build

2. `scripts/start_production.sh`

Should:

* build frontend if `frontend/dist` does not exist
* start FastAPI with host `0.0.0.0`
* use `$PORT` if available, default to `8000`

Example:
`uvicorn backend.app.main:app --host 0.0.0.0 --port ${PORT:-8000}`
