You are building a full-stack project.

Project name:
**SFT Dataset Curation Workbench**

---

# 🧠 Product Goal

Build a platform to:

* Inspect already-distilled SFT datasets
* Apply filtering step-by-step
* Visualize every transformation
* Export filtered datasets

IMPORTANT:

* This platform DOES NOT generate distillation data
* It ONLY works on already-generated datasets
* It must be fully transparent and inspectable

---

# ⚠️ CORE REQUIREMENT (MOST IMPORTANT)

Every operation must create a new **stage**.

DO NOT overwrite data.

Pipeline structure:

Stage 0: raw dataset
Stage 1: after filter A
Stage 2: after filter B
...

Each stage must be independently inspectable.

---

# 🔁 Stage Requirements

Each stage MUST provide:

* rows_kept
* rows_removed
* removal_reason (for each row)
* input_count
* output_count
* removed_count
* filter_config
* summary statistics
* distributions

User MUST be able to:

* inspect any stage
* compare with previous stage
* understand exactly what changed

---

# 📦 Input Dataset

User provides:

* JSONL dataset (distilled SFT data)
* hacking removal script
* deduplication script

Dataset may contain:

* prompt
* response
* source_model
* signature
* problem_type
* behavior_type
* correctness
* runtime_ms
* compiled
* hacked
* duplicate
* metadata

Fields may be missing → handle safely.
