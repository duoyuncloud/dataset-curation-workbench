Update subset-based curation with simplified dimensions.

Current state:
- Task-based persistence exists.
- Signature subset selection already exists.
- Filters exist.
- We want to simplify subset dimensions for MVP.

Goal:
Subset selection should support only:
1. signature
2. data_stage

Important terminology:
- data_stage = optimization stage extracted from the question text, e.g. Stage 1 / Stage 2 / Stage 3
- platform stage = filtering pipeline stage, e.g. stage_0 raw, stage_1 after dedup

Do NOT confuse these two.

---

# 1. Rename/clarify stage field

Each data row should have:

- signature
- data_stage

Do not call the row-level extracted stage just "stage" in backend/frontend code, to avoid confusion with platform stages.

Platform stages remain:

- stage_0
- stage_1
- stage_2
- etc.

---

# 2. Extract data_stage from question

On upload / stage creation, extract data_stage from question using regex:

Stage\s*(\d+)

Examples:
- "Stage 3: Shared Memory Caching" → data_stage = "3"
- "Current Stage Plan: Stage 2" → data_stage = "2"
- no match → data_stage = "unknown"

Store data_stage as a string.

# 4. Subset selector

Update subset selector above data table to support:

- signature
- data_stage

User can select either or both.

Examples:
- signature = conv

Subset selection is view-only and must NOT create a new platform stage.

---

# 5. Apply filters to subset

If a subset is active and user applies filters:

- apply selected filters only to matching subset rows
- keep non-subset rows unchanged
- create a new platform stage
- write full updated kept.jsonl for the new stage
- write removed.jsonl for rows removed from the subset
- record subset_filter in platform stage metadata

Example:
Current platform stage has 10,000 rows.
Active subset: signature=conv, data_stage=3 → 1,000 rows.
Filter removes 200 rows.
New platform stage has 9,800 rows.
Other 9,000 rows are unchanged.

# 8. Balancing filters

Add two distribution filters:

## RandomDropFilter

Config:
- drop_fraction: float between 0 and 1
- random_seed: int

Behavior:
- randomly remove that fraction from the current target rows
- if subset is active, only drop from subset
- if no subset, drop from full current stage

Removed rows include:
- removal_reason = "random_drop"
- drop_fraction
- random_seed

## BalanceToMeanFilter

Config:
- group_by: "signature" or "data_stage"
- random_seed: int

Behavior:
- compute group counts within the current target rows
- compute mean count
- groups above floor(mean) are randomly downsampled to floor(mean)
- groups below mean remain unchanged

Removed rows include:
- removal_reason = "balance_to_mean"
- group_by
- group_value
- original_group_count
- target_count
