# 2. File storage abstraction

Create/extend StorageBackend:

class StorageBackend:
    save_text(path: str, content: str) -> None
    load_text(path: str) -> str
    exists(path: str) -> bool
    delete_prefix(prefix: str) -> None
    save_jsonl(path: str, rows: list[dict]) -> None
    load_jsonl(path: str) -> list[dict]
    save_json(path: str, obj: dict) -> None
    load_json(path: str) -> dict

Implement:

1. LocalStorageBackend
- uses DATA_DIR
- current local behavior

2. S3StorageBackend
- supports AWS S3 / Cloudflare R2 / any S3-compatible storage
- use boto3

Environment variables:

STORAGE_BACKEND=local | s3

For local:
DATA_DIR=./data

For s3:
S3_BUCKET=
S3_ENDPOINT_URL=
S3_ACCESS_KEY_ID=
S3_SECRET_ACCESS_KEY=
S3_REGION=auto

If STORAGE_BACKEND=s3, validate required env vars at startup.

# 3. File paths

Keep logical paths the same:

tasks/{task_id}/raw/input.jsonl
tasks/{task_id}/stages/stage_0/kept.jsonl
tasks/{task_id}/stages/stage_0/removed.jsonl
tasks/{task_id}/stages/stage_0/summary.json
tasks/{task_id}/stages/stage_0/distribution.json

For local:
DATA_DIR/tasks/{task_id}/...

For s3:
s3://S3_BUCKET/tasks/{task_id}/...

Database should store logical storage paths, not absolute local paths.
